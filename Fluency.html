Part Two · From Theory to Execution

From Skill to Practice

The first guide named the four competencies. This one turns them into a single connected workflow — four simple frameworks where each decision feeds the next, so a choice you make at the start still shapes what you check at the end.

Continues from “From Stack to Skill” — you don't need it first, but the 4Ds live there.
Four frameworks ~11 min read One handoff chain
00 — The Idea

Fluency is a relay, not four separate skills

Most people meet the 4Ds as a checklist: four things to be good at. That is true but incomplete. In practice they behave like a relay — each stage produces something small and specific that the next stage runs on. Get the handoff right and the whole chain gets easier. Get it wrong at the start and no amount of clever prompting downstream can recover it.

So the useful question is not “how do I do Description well?” in the abstract. It is: given the call I made at Delegation, what does good Description now look like — and what does that, in turn, tell me to check? That single thread is what this piece builds. Here is the chain in one view.

Four short frameworks follow — one per stage. Each ends by handing you the exact artefact the next one needs.

01 — Delegation

The Leverage Scan: find the work worth handing over

Delegation fails most often not because people pick the wrong tool, but because they never looked at their work as a whole before reaching for one. The Leverage Scan fixes that. It is a two-minute pass over any task or week that sorts work by a single question: where does AI create the most value for the least risk?

Run every candidate task through two axes. Leverage — how much time, reach or quality is on the line if this goes faster or better. Suitability — how well-matched the task is to what AI does reliably (bounded, text- or pattern-heavy, tolerant of a draft-then-refine loop). Plot the two and the quadrant tells you what to do.

The scan does one more thing, and it is the important one: for each task you keep, it records three facts that become the task profile you hand to Description. What mode you're in (automation, augmentation or agency), how much is at stake, and what "done" would actually look like.

Then pick the surface

Choosing to use AI isn't one choice — it's two. The second is where. The same task handled in a chat window, a delegated-work surface, or a coding environment gives you very different leverage, and the right pick falls out of the profile you just built. Read across from the mode and the shape of the deliverable, and the surface chooses itself.

Shape · a conversation

Chat assistant

You stay in the loop, turn by turn.

Thinking, drafting, analysing, quick Q&A. You supply context and steer every step.

Pick when: mode is augmentation and you want a collaborator you actively drive.→ Claude chat
Shape · a job to run

Agentic workspace

You set a goal; it works across steps.

Multi-stage research, analysis over many files, tasks that run with light supervision and hand back a package.

Pick when: mode is agency, or automation spanning many steps.→ Claude Cowork
Shape · code & systems

Coding environment

It reads, writes and runs a codebase.

Building, debugging, refactoring, shipping. The deliverable is software that has to actually run.

Pick when: the artefact is code — in any mode.→ Claude Code
Shape · inside a document

Embedded surface

The AI meets you in the file itself.

Work that lives in a spreadsheet, a deck, a doc or the browser — where leaving the tool would break the flow.

Pick when: the output is the document you're already in.→ Claude in Excel · PowerPoint · Chrome

The named tools are today's Claude surfaces; the four shapes are the durable part — any assistant, agent, coding tool and embedded feature maps onto them. Match the shape, not the brand, and the choice survives the next product release.

Choosing to use AI is really two choices: what kind of task it is, and which surface it belongs on. The scan settles both before you type a word.

02 — Description

The Spec Ladder: match how you ask to what you delegated

Here is where the chain earns its keep. Description isn't one skill applied uniformly — the kind of description a task needs is set by the mode you recorded at Delegation. A bounded automation task and an open agency task need very different instructions. The Spec Ladder maps mode to method so you stop over- or under-specifying.

If mode =Automation

Describe tightly. Precision beats freedom. The task is bounded, so pin the output shape, the format and the constraints. Lead with product description — exactly what you want back — and give an example of “good.”

"Extract every dollar figure from this PDF into a two-column table: item, amount. Numbers only, no commentary."

If mode =Augmentation

Describe the process, then iterate. You're collaborating, so set the role and the thinking method, not just the output. Lean on process and performance description and expect a back-and-forth.

"Act as a sceptical editor. Read my draft, then think step by step about its three weakest claims before suggesting fixes."

If mode =Agency

Describe the goal, the guardrails and the stop conditions. The AI will make choices you won't see, so specify success, boundaries, and when to pause and ask. This is spec-writing, not prompting.

"Goal: draft replies to these 12 emails. Never send. Flag anything involving money or legal terms for me. Stop and ask if unsure."

Notice that “prompting” — the phrasing everyone obsesses over — is only the surface of the automation rung. As you climb toward agency, description becomes less about wording and more about defining a contract: what success is, what's off-limits, and where the AI must hand control back. That contract is also, conveniently, your success test — which is exactly what Discernment needs.

Description producesA spec with a success test: the instruction, plus an explicit statement of what “right” looks like.
→
Discernment receivesA standard to check the output against — written before the output existed, so it can't be swayed by it.

If you can't state the success test while you're writing the prompt, you're not ready to judge the answer. Write the test first.

03 — Discernment

The Check Matches the Task: right-sized evaluation

The mistake at this stage is checking everything the same way — either rubber-stamping a high-stakes output or laboriously fact-checking a throwaway one. The type of evaluation should be set by the same two facts you've been carrying since Delegation: the mode and the stakes. The Spec Ladder told the AI what to do; now the same profile tells you how hard to look.

The first article split discernment three ways — product, process and performance. Here's when each earns your attention, and how deep to go.

AutomationSpot-check the product

The output is bounded and testable, so check the result against a known truth. Verify a sample, confirm the format, look for the confidently-invented number. You rarely need to inspect its reasoning.

Recompute three rows of the extracted table by hand. If they match, trust the rest.

AugmentationInterrogate the process

You care how it got there, because you'll build on it. Read the reasoning, not just the conclusion. Did it follow the method you set? Did it skip a step, or smuggle in an assumption?

"You recommended option B — walk me through what you ruled out and why." Then pressure-test that.

AgencyAudit the trail

The AI acted across steps you didn't watch, so evaluate the whole trajectory, not just the final artefact. Check what it touched, what it changed, and every point where it made a call. Sample the actions, not only the outcome.

Before accepting 12 drafted replies: read all 12, confirm none were sent, verify the money/legal flags fired correctly.

The rule of thumb: as autonomy rises, your check moves from the answer to the journey. Low-autonomy work lets you verify the product cheaply. High-autonomy work forces you to audit the process, because the product alone can hide a bad decision made three steps back. Whatever survives this check is the verified result you now have to stand behind — which is where Diligence begins.

Discernment producesA verified result — plus a note of what you checked and how confident you are.
→
Diligence receivesSomething you've judged fit to use — and now must decide how far you're willing to send.

Don't check every task the same way. Check it the way its autonomy demands — the answer for bounded work, the whole journey for autonomous work.

04 — Diligence

The Exposure Ladder: match diligence to who's affected

Diligence is where the chain closes. Everything upstream produced a result you trust for yourself; diligence asks a different question — who else is exposed if this is wrong, and what do you owe them? The single best predictor of how much diligence a task needs is not how clever the AI was, but how far the output travels and who relies on it.

So sort by exposure, not by effort. The further a piece of work travels from you, the higher the rung — and each rung adds a non-negotiable gate before release.

LVL 11

Personal & exploratory

Audience: just you · reversible

Drafts, brainstorms, learning, private analysis. A mistake costs only your own time.

Gate: a sanity read. Use freely; trust but verify anything you'll carry forward.

LVL 22

Internal & shared

Audience: your team · colleagues act on it

Team reports, internal memos, analysis others will build decisions on. Errors propagate to people who trusted you.

Gate: verify key facts and figures · be transparent that AI was involved so others can weight it.

LVL 33

External & attributed

Audience: customers, public · your name on it

Client deliverables, published content, anything carrying your or your organisation's reputation. Errors become your errors, publicly.

Gate: full fact-check · human sign-off · no confidential data used · disclosure where it matters.

LVL 44

Regulated & consequential

Audience: government, industry, safety-critical

Regulatory filings, legal, medical, financial advice, anything where a person could be harmed or a law engaged.

Gate: everything above, plus qualified expert review · documented accountability · AI treated as assistant, never author of record.

The ladder makes the hardest judgement — “is this a big deal?” — mechanical. You don't estimate risk in the abstract; you ask a concrete question: who receives this? The answer picks the rung, and the rung names the gate. And because you tagged the stakes back at Delegation, a Level 3 or 4 task should never surprise you here — you'll have seen it coming from the very first step.

Diligence producesA release decision you can defend — the right gate cleared for the right audience.

05 — Try It

The workflow router: make one call, watch it cascade

This is the whole argument in one control. Make the two decisions you'd make at Delegation — pick a mode and an exposure level — and the frameworks resolve into a single, task-specific plan: the surface to reach for, how to describe, how to check, and what gate to clear. Change either input and watch every downstream stage update. That propagation is the point: the frameworks are connected, not separate.

06 — End to End

One task, all four stages

To see the handoffs carry, follow a single ordinary task the whole way down the chain.

The task

“Summarise 40 customer reviews into a one-page insight brief for next week's leadership meeting.”

01 Delegation

Leverage Scan: high value, strong fit → Automate first. Mode = augmentation. Surface = chat (you'll drive the angle). Stakes = internal. Profile handed over.

02 Description

Augmentation rung: set the role and process. “Cluster the reviews into themes, show me the clusters, then we'll pick the three worth briefing.” Success test written.

03 Discernment

Augmentation → interrogate the process. Read the clustering logic; trace two quotes back to real reviews to catch invented sentiment. Verified.

04 Diligence

Exposure = internal/shared → Level 2. Gate: verify the headline figures, note AI helped draft it. No client data leaves. Cleared to send.

No stage was done in isolation. The augmentation call at Delegation is why Description set a process and why Discernment read the reasoning. The internal audience is why Diligence stopped at a Level 2 gate rather than a full external fact-check. Change the task to “publish these insights in our public quarterly report” and only two inputs move — exposure jumps to Level 3 — but the whole downstream plan tightens automatically. That's the relay working.

In closing

The first guide argued that prompting is a quarter of one competency, and that the real skill is orchestration. This is what orchestration looks like up close: not four things you're separately good at, but one chain you run deliberately — a call at the top that sets the ask, the check and the gate all the way down. Master the handoffs and you stop treating each task as a fresh puzzle. You just run the relay, and it carries you from a rough idea to a result you can stand behind.
